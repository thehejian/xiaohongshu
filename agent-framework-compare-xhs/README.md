## 📖 文章正文

N8N · Dify · Coze(扣子) 怎么选？

四个主流 AI Agent 框架，各有侧重。看完这张图，选对工具不踩坑 👇

---

### 1️⃣ N8N — 工作流自动化王者

· 开源可自托管 · 节点式编排 · 500+ 集成

N8N 的核心优势是**工作流自动化**。拖拽节点就能串联 API、数据库、AI 模型。适合做：自动化脚本、数据同步、内部工具集成。

自托管免费，数据可控，开发者最爱。但 LLM 原生能力弱，需要自己接 API。

**适合谁**：有开发能力、重视数据隐私的团队

---

### 2️⃣ Dify — LLM 应用开发平台

· 开源 · RAG + Agent + 工作流 · 模型管理

Dify 定位**LLM 应用开发平台**。内置 RAG 知识库、Agent 工作流、模型路由管理。写 Prompt、调模型、建知识库，一站式搞定。

开源可自部署，社区活跃。比 N8N 更懂 LLM，比 Coze 更灵活。

**适合谁**：想快速搭建 LLM 应用、需要知识库的企业

---

### 3️⃣ Coze / 扣子 — 字节出品，插件生态强

· 闭源 · 插件市场 · 多模态 · 一键发布

Coze（国内版叫**扣子**）是字节跳动的 AI Bot 平台。最大优势是**插件生态**——上百个现成插件，数据库、搜索、绘图都能用。

支持多模态（文本+图片+语音），一键发布到抖音/飞书/微信。闭源 SaaS，上手最快。

**适合谁**：想快速上线 Bot、依赖插件生态的个人/小团队

---

### 📊 对比总览

| 框架 | 核心优势 | 开源 | 上手难度 |
|------|---------|------|---------|
| N8N | 工作流自动化 | ✅ | 中等 |
| Dify | LLM 应用开发 | ✅ | 中等 |
| Coze/扣子 | 插件生态 + 多模态 | ❌ | 低 |

---

### 🎯 怎么选？

- 做自动化流程 → **N8N**
- 做 LLM 应用 + 知识库 → **Dify**
- 做 Bot + 快速上线 → **Coze/扣子**
- 三者组合用也不冲突（N8N 做底层自动化 + Dify 做 LLM 层 + Coze 做前端交互）
