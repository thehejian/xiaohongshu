# 003-Twitter 小红书内容生产流水线

## 一、项目定位
- 社交媒体系列：Markdown 文章 + PNG 卡片，投放于 **小红书 / 微博**
- 两种内容类型：**文章型**（单品深挖）和 **排行榜型**（TOP10 盘点）
- 当前 19 个内容文件夹 + twitter/ 实验性侧线

## 二、核心工具链（6 件套）

| 工具 | 用途 | 关键约束 |
|------|------|----------|
| **Python 3** (stdlib only) | SVG 生成、字符计数、文本处理 | 无外部依赖 |
| **Inkscape 1.4+** | SVG → PNG 栅格化 | 必须显式像素尺寸，不支持 `100%` |
| **opencli** | 浏览器自动化发布小红书草稿 | `--draft true`, `--window foreground` |
| **lark-cli** | 飞书文档管理（预览用） | `+create`, `+media-insert` |
| **Git** | 版本控制 + 发布前备份 | push before publish |
| **浏览器桥接扩展** | opencli 控制真实 Chrome | 必要时用 DataTransfer 注入绕过 upload 缺陷 |

## 三、完整生产流水线（6 步）

```
待办.txt 选题队列
    ↓
① 调研与撰稿
   ├── 网络抓取 / 收集数据
   └── 产出: article.md (XHS 草稿) + README.md (完整博客)
    ↓
② 卡片生成
   ├── 编写 gen_cards.py (Python f-strings 构建 SVG)
   ├── 运行 python3 gen_cards.py
   ├── 输出 .svg → Inkscape 子进程 → .png
   └── 尺寸: 1024×1024(封面) / 800×800 / 1792×1024(横幅)
    ↓
③ 飞书预览
   ├── lark-cli docs +create
   ├── lark-cli docs +media-insert (每张卡片)
   └── 验证效果
    ↓
④ Git 备份
   └── git add + commit + push
    ↓
⑤ 小红书发草稿
   └── opencli xiaohongshu publish --draft true
    ↓
⑥ 用户手动审核发布
   └── 登录 creator.xiaohongshu.com → 草稿箱 → 发布
```

## 四、每篇内容的标准产出物

- **article.md** — 标题（≤20 字）+ 正文（≤950 字）+ 图片映射 + 话题标签
- **README.md** — 完整博客版（含文件清单 + 来源）
- **gen_cards.py** — Python SVG 生成脚本
- **N 张 SVG** — 卡片源文件
- **N 张 PNG** — 最终渲染图（≤9 张/帖）

## 五、两种设计语言

| 风格 | 背景色 | 适用场景 | 示例 |
|------|--------|----------|------|
| **浅色 (推荐)** | 奶油色 `#FAF7F2` | 小红书友好、通用 | cc-switch, ai-rice-bowl |
| **深色** | 深黑 `#0A0A14` + 径向渐变 | 极客向产品深挖 | hermes, minimax-m3 |

## 六、小红书内容约束

- 标题 ≤20 字符（最佳 14-18）
- 正文 ≤1000 字符（习惯 950）
- 图片 ≤9 张/帖
- 话题标签：逗号分隔、不用 `#` 前缀
- 禁止：首先/其次/最后/总之、绝绝子/yyds/炸裂/神器

## 七、已知坑点（MEMORY.md 记录）

- Inkscape 对 CJK `letter-spacing` 渲染异常 → 改用英文替代
- `height="{h}>` 缺引号 → 静默输出空白 PNG
- opencli 1.8.0/1.8.1 `upload` 命令的 `SyntaxError` → 改 `const` 为 `var`
- lark-cli `+media-insert` 只能追加末尾，不能插中间
- 发帖前需用 PIL + numpy 检测 blank PNG
