# 003-Twitter 小红书内容生产流水线

## 一、项目定位
- 社交媒体系列：Markdown 文章 + PNG 卡片，投放于 **小红书 / 微博**
- 两种内容类型：**文章型**（单品深挖）和 **排行榜型**（TOP10 盘点）
- 当前约 46 个 `-xhs/` 内容文件夹：文章型 + 排行榜型
- Twitter/X 发布见独立文件 `twitter生产流水线大纲.md`
- 仓库：`https://github.com/thehejian/xiaohongshu.git`

## 二、核心工具链

| 工具 | 用途 | 关键约束 |
|------|------|----------|
| **Python 3** (stdlib only) | SVG 生成、字符计数、文本处理 | 无外部依赖 |
| **Inkscape 1.4+** | SVG → PNG 栅格化 | 必须显式像素尺寸，不支持 `100%` |
| **opencli** | 浏览器自动化发布小红书草稿 | `--draft true`, `--window foreground`, `--site-session persistent`, `-f yaml` |
| **lark-cli** | 飞书文档管理（预览用） | `+create`, `+media-insert`, `+update --mode overwrite` |
| **Git** | 版本控制 + 发布前备份 | push before publish |
| **浏览器桥接扩展** | opencli 控制真实 Chrome | 每次会话前确认扩展已连接 |
| **opencli gemini image** | AI 生成封面/插图 | `--rt 1:1` 方图；替代方案：opencli + Puppeteer 渲染 |
| **Puppeteer-core** (按需) | HTML→PNG 渲染多页卡片 | 需本地 Chrome，`npm install puppeteer-core` |

## 三、完整生产流水线（6 步）

```
待办.txt 选题队列
    ↓
① 调研与撰稿
   ├── 网络抓取 / 收集数据（webfetch/GitHub API curl）
   └── 产出: article.md (XHS 草稿) + README.md (完整博客)
    ↓
② 封面/AI 插图（可选）
   └── opencli gemini image "<prompt>" --rt 1:1
    ↓
③ 卡片生成
   ├── 编写 gen_cards.py (Python f-strings 构建 SVG)
   ├── 运行 python3 gen_cards.py
   ├── 输出 .svg → Inkscape 子进程 → .png
   ├── 尺寸: 1024×1024(封面) / 800×800 / 1792×1024(横幅) / 1242×1660(3:4)
   └── 立即验证: file size < 50KB → 空白，需检查 SVG 引号闭合
    ↓
④ 飞书预览
   ├── lark-cli docs +create (markdown 不含 H1)
   ├── lark-cli docs +media-insert × N (每张图一次，相对路径)
   └── lark-cli docs +fetch 验证 token 数
    ↓
⑤ Git 备份
   └── git add + commit + push
    ↓
⑥ 小红书发草稿 / 直接发布
   ├── 草稿: opencli xiaohongshu publish --draft true
   └── 直接发布: 去掉 --draft true
    ↓
⑦ 用户手动审核发布
   └── 登录 creator.xiaohongshu.com → 草稿箱 → 发布
```

## 四、每篇内容的标准产出物

- **article.md** — 标题（≤20 字）+ 正文（≤950 字）+ 图片映射 + 话题标签
- **README.md** — 完整博客版（含文件清单 + 来源）
- **gen_cards.py** — Python SVG 生成脚本
- **N 张 SVG** — 卡片源文件
- **N 张 PNG** — 最终渲染图（≤9 张/帖）

## 五、卡片设计语言（三种风格）

| 风格 | 背景 | 适用场景 | 示例 |
|------|------|----------|------|
| **浅色 v1（推荐）** | 奶油色 `#FAF7F2` → `#F3F0E8` 渐变 + 白色卡片 | 小红书友好、通用、多 app 对比 | cc-switch, ai-rice-bowl, api-price-war, ai-ppt-tools |
| **深色** | 近黑 `#0A0A14` + 径向渐变 | 极客向产品深挖、品牌色为深色 | hermes, minimax-m3, opencli, OmO |
| **v3 高饱和全幅渐变（新）** | 整张 1024×1024 满版渐变，无内白卡 | 爆款封面、品牌联名、需要视觉冲击力 | hermes-feishu-xhs |

### v3 设计要点
- **全幅渐变背景**：没有内白卡，文字直接写在渐变上
- **双 Logo 布局**：左右对称放置品牌 logo（白色圆底 + shadow），中间 🤝 握手图标
- **半透明底栏**：`rect fill="#000000" opacity="0.15"` 叠加在渐变上方，承载描述文字
- **大数字编号**：01-07，140px，白→金黄→白渐变填充（`url(#titleGrad)`）
- **无文字投影**：用户明确要求文字不要 glow filter，只保留 logo 白圆底座的 shadow filter
- **装饰星点**：✦ 字符作为简约装饰，比圆更轻盈

**卡片填充经验**：每张卡片只传达一个信息，不堆叠。避免图表/柱状图/拓扑图，保持纯文字 + 简洁装饰。

## 六、小红书内容约束

- 标题 ≤20 字符（最佳 12-18 字，`echo -n "标题" \| wc -m`）
- 正文 ≤1000 字符（习惯 950，`wc -m` 不带换行）
- 图片 ≤9 张/帖
- 话题标签：逗号分隔、不用 `#` 前缀
- **正文第一行 = 标题**（用户偏好，标题在正文开头重复）
- **正文不含时间/版本号**：去掉日期、版本号、commit 数
- **纯文本格式**：小红书不支持 Markdown（`###`、`---`、`**` 会显示为原文）
- 禁止：首先/其次/最后/总之、绝绝子/yyds/炸裂/神器

## 七、已知坑点（详见 MEMORY.md）

### SVG 致命坑
- `<height="{h}>` 缺闭合 `"` → Inkscape 静默输出空白 PNG（文件 <50KB）
- `<rect width="100%">` 不被 Inkscape 支持 → 必须显式 `width="1024"`

### Inkscape 中文渲染
- CJK `letter-spacing` 与某些字符冲突（"评测""发布"方块）→ 改用英文替代
- `<text>` 不支持 `\n` 换行 → 必须用 `<tspan x="..." dy="...">`

### opencli 发布
- `--window foreground` 必须（后台无法上传图片）
- `--site-session persistent` 避免重复扫码
- 图片路径必须相对 CWD（绝对路径报 `Image file not found`）
- 每次 `--draft true` 都新建草稿，不覆盖旧草稿
- content 含换行时用 `"$(cat /tmp/body.txt)"` 传递
- **对 niche 主题不要传 `--topics`**（2026-06-05 验证）：Obsidian/AI Agent 等小众话题 attach 失败，报 `no real topic entity appeared`。解决方案：body 末尾用 `#话题` 标签替代，省略 `--topics` 参数
- **发布状态有三种返回值**：
  - `✅ 发布成功` — 自动完成发布（无 `--draft` 时）
  - `✅ 暂存成功` — 草稿已保存（有 `--draft true` 时）
  - `⚠️ 操作完成，请在浏览器中确认` — 图片已上传但需用户在浏览器手动点发布
- 1.8.0/1.8.1 `upload` 命令有 `SyntaxError` → 改 `const` 为 `var` 修复

### lark-cli
- `+media-insert` 只能追加末尾，不能插中间 → 所有图片放文末"图集"章节
- `--file` 参数是单数，路径必须相对 CWD
- `+update --mode overwrite` 清空文档含 media blocks，需重插全部图片
- `opencli gemini image` 不可靠（连续 4+ 次返回 `no-images`），SVG gen_cards.py + Inkscape 是唯一可靠的封面生成方式

### 卡片设计
- **先算后画**：用 Python `text_extent()` 检查相邻元素 gap ≥ 10px 安全
- **X 轴分离优先**：文字和 icon 不要在 Y 轴叠放，尽量用 X 轴分离
- `<text>` y 属性是基线不是视觉中心，单行视觉中心偏移 ≈ fontsize * 0.15
- **图片生成后立即验证**：`PIL.Image` → `np.unique(colors)` < 10 → 空白
- 生成后 `ls -la` 检查文件大小：1024×1024 > 400KB 正常，< 50KB 空白
- **SVG 中禁止使用 HTML entity**：`&middot;`（·）和 `&rarr;`（→）不是 XML 标准实体，Inkscape 会报 `parser error: Entity not defined` 并渲染空白。必须用 Unicode 字面量 `·` (U+00B7) 和 `→` (U+2192) 写入 f-string
- **Logo 获取**：从官网下载 favicon（`/favicon.ico`）或 CDN 图标，PIL 放大到 128×128，base64 存 `.json` 文件，运行时读取
- **文字不用 glow filter**：用户反馈投影影响可读性，用纯色文字 + 高对比度背景代替
- **v3 cover 双 Logo 布局**：左右对称 + 中间握手图标，底部安装命令 + badge

### 卡片样式严格规范（2026-06-05 新增）

**浅色卡片（中文，小红书用）：**
```
底色：#FAF7F2（纯色，无渐变）
文字定位：绝对居中 x="50%" y="50%" text-anchor="middle" dominant-baseline="middle"
字号：封面 100-110px，内容卡 68-80px
颜色：深色 #1E293B + 品牌强调色（粉 #E63E6B / 绿 #0D9488 / 橙 #F59E0B / 紫 #8B5CF6）
禁止：白色文字（#FFF/#FFFFFF）、灰色文字、投影/阴影/滤镜
删除：标题栏、底部辅助文字、大块白色色块
约束：字不重叠、不溢出、纯扁平化
```

**深色卡片（英文，Twitter 用）：**
```
底色：#0A0A14（纯色，无渐变）
文字：#F5F5F7（主色）+ #C4B5FD（紫）/ #67E8F9（青）/ #FCD34D（黄）/ #F472B6（粉）
定位、字号、无阴影、无标题/底部文字规则与浅色相同
禁止：黑色、深灰色、暗色/泥泞色文字
```

**解决方案聚焦原则：**
- 4 张卡片必须包含一张专属的"解决方案"卡（如"3个解药"/"3 Antidotes"）
- 解决方案卡使用最大字号（96px）和最醒目的强调色（金色/琥珀色）
- 排列顺序：封面（问题陈述）→ 后果卡 → **解决方案卡**（核心焦点）→ 行动号召卡

**双语言卡片设计模式（2026-06-05 obsidian-agent-xhs 验证）：**
- 同一主题可同时生成浅色中文 + 深色英文两套卡片
- 一套 gen_cards.py 同时包含 light + dark 两组函数，共享颜色变量
- 内容对应但不完全相同：中文走痛点→方案→数据线，英文走哲学→架构→结论线
- 中文封面：更大字号（96px），更情绪化，平台标签
- 英文封面：略小字号（80px），更理性，强调争议/认知感
- 中英文封面各自用 1024×1024，内容卡用 800×800

**合集型内容卡片密度规范（2026-06-05 ai-tool-stack-2026-xhs 验证）：**
- 每张卡片必须有具体条目（工具名 + 简短点评），不能只有大标题
- 每张卡片 4-7 个具体条目，使用垂直列表或 2×2 网格布局
- 分类标题 + 具体条目组合结构
- 条目太多时拆分为多张卡片（每张 3-6 条），共 4 张

### XHS `--topics` 对 niche 主题必须省略（2026-06-05）
- `--topics` 参数需要 XHS 系统能匹配到真实话题实体
- 对 niche/小众主题（如 "Obsidian""AI Agent"），topic attach 会失败 → `no real topic entity appeared`
- 解决方案：body 末尾用 `#话题` 标签替代 `--topics`，发布时直接省略 `--topics` 参数

### Puppeteer 截图
- `page.screenshot({ selector })` 截取固定视口 → 用 `element.screenshot()` 截元素本身

## 八、爆款写作规范

### 调性要求（严禁平铺直叙）
- **利他价值 + 情绪共鸣**：让读者觉得"这对我有用"或"这就是在说我"
- **高互动感口吻**：姐妹们 / 亲测 / 避坑 / 大数据帮我推给 / 听我一句劝
- **个人故事钩子**：用"我本来想...结果..."的叙事结构，不是"AI 面试模拟是什么"
- **口语化**：像朋友聊天，不是写论文

### 结构模板（痛点引入 + 干货/金句 + CTA）
```
标题（≤20字，含数字/悬念/痛点）
    ↓
个人故事钩子（"我本来想报8000块的班，结果..."）
    ↓
痛点共鸣（"结构化面试最怕什么？不是你不会答，是你当场大脑空白"）
    ↓
干货/解决方案（"AI 面试官真的狠，专挑逻辑漏洞往死里挖"）
    ↓
行动呼吁（"别等面试前一周才想起来练，现在就开始"）
    ↓
金句收尾（"一杯奶茶换个铁饭碗，这账怎么算都不亏"）
    ↓
话题标签（4-6个）
```

### 标题公式（含数字/悬念/痛点）
- 情绪式：`姐妹们！49块AI面试模拟亲测真香了😭`
- 悬念式：`考公面试班8000块，我选了49块的AI，结果是...`
- 对比式：`8000块 vs 49块，考公面试怎么选？`
- 结论式：`AI面试模拟，比报班还香`

### 正文格式
- 标题 ≤ 20 字，必须作为正文第一行
- 正文 ≤ 950 字，无时间戳
- 短段落（1-2行），密集 emoji 分段
- 不用"首先/其次/最后/总之"（AI 味重）
- 不用营销词（"重磅""炸裂""神器"）
- 数据点用 `·` 分隔

## 九、合集/排行榜型内容生产规范

### 适用场景
- 30个/10个/Top 榜单类文章（如"30个免费AI神器合集"）
- 需要在多张卡片间分配大量条目

### 卡片内容密度规范
- **每张卡片必须有具体条目**（工具名 + 简短点评），不能只有大标题
- 分类标题 + 6条以内具体内容，3×2网格或垂直列表布局
- 条目太多时拆分为多张卡片（每张3-6条），共4张

### 浅色/深色语言规则
- **浅色卡片 → 中文**：奶油底渐变 `#FAF7F2→#F5F0E8`，文字 `#1E293B`
- **深色卡片 → 英文**：深蓝底 `#0B1027`，文字 `#FFFFFF` 或高亮色
- 不可混用：浅色配英文或深色配中文都造成风格不一致
- 浅色卡无白色/灰色文字；深色卡无黑色/暗色文字

### SVG 特殊字符
- `&` 必须写 `&amp;`，否则 Inkscape 报错 `xmlParseEntityRef: no name`
- 中文中避免 `&`，如 `"Free & Open"` 改为 `"Free and Open"`

### 卡片布局
- 橙色标题条 `#E85D04` + 白色文字，位于顶部
- 标题条高度 100px，下方 10px 间距排内容
- 无文字投影/阴影，纯扁平化
