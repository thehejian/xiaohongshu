# Claude 高阶提示词 8 招 —— 复制即用的高级 Prompt 🔥

> 小红书风格文章 · 正文 ~820 字
> 灵感来源：Anthropic 官方文档 + 长期实战经验

---

## 📖 文章正文

用 Claude 大半年，踩过无数坑。从"AI 回答太水"到"输出越来越准"，关键就在于**提示词的写法**。

下面 8 条是实测最有效的**高级 Prompt 模式**——不是泛泛而谈的原则，是**每条都带模板**的硬核技巧 👇

---

### 1️⃣ 结构标签法 —— 给 Claude 划边界

Claude 对 XML 标签的理解远超其他模型。用标签把输入结构化，输出质量直线上升。

**模板：**
```
<task>写一篇小红书文案</task>
<context>受众是 AI 入门新手</context>
<format>不超过 900 字，不加 markdown</format>
<examples>...</examples>
```

**原理：** Anthropic 官方文档明确推荐 `<tags>` 方式。标签就像给 Claude 画了一个"框"，它不会跑偏。

---

### 2️⃣ 反向追问法 —— 先澄清再回答

Claude 最怕什么？**猜错你的意图后硬答。**

**模板：**
> "在回答之前，先向我提 3 个澄清问题。我回答后，基于我的回答再出最终方案。"

**原理：** 相当于让 Claude 先做需求确认，避免"做完了才发现理解错了"的悲剧。

---

### 3️⃣ 思考链法 —— 看得到推理过程

适合写分析、决策、代码调试。

**模板：**
> "请逐步推理。每一步都要标注依据（数据来源 / 逻辑推导 / 代码输出）。最后再给出结论。"

**原理：** Claude 的"思考"输出不消耗额外 token（对 Claude Code / API 而言是 extension），而且你能看到它的推理路径，方便修正方向。

---

### 4️⃣ 角色锚定法 —— 身份越具体越好

对比实验：
- ❌ "你是一个 PM" → 输出平庸
- ✅ "你是字节跳动 10 年经验的资深产品经理，带过 DAU 5000 万的产品，说话直接犀利，讨厌废话。"

**模板：**
> "你是 [具体公司/角色/经验年限/风格] 的 [身份]。[补充性格特征/偏好]。"

**原理：** 具体角色能激活 Claude 的"角色扮演"能力，风格差异极其明显。

---

### 5️⃣ 示例驱动法 —— Few-Shot 永远最稳

**模板：**
> "我给你 3 个例子。例1 是[场景]的输出，例2 是[场景]的输出，例3 是[场景]的输出。请按同样的风格和格式回答我的新问题。"

**原理：** Few-shot prompting 在所有研究文献中都被证明是最稳定提升质量的手段。Claude 对格式模仿尤其擅长。

---

### 6️⃣ 输出锁定法 —— 省去后处理

**模板：**
> "只输出 JSON。key 为：title, content, tags。不要多余文字、不要解释。"

**原理：** Claude Sonnet 4 / Opus 3.5 的 JSON 输出可靠性极高。再加一层 `--format json` 或 system prompt 强化，解析零故障。

---

### 7️⃣ 对抗审查法 —— 自找麻烦

适合方案评审、代码审查、PRD 评估。

**模板：**
> "请用最严格的标准审查我的方案。指出 3 个盲点、2 个假设缺陷、1 个你不同意的地方。不要客气。"

**原理：** Claude 默认"讨好"用户。你必须主动要求它**批评你**，才能得到真正的洞见。

---

### 8️⃣ 知识边界法 —— 给它说谎豁免权

**模板：**
> "如果不知道就直说不知道。只基于我给你信息和知识库的内容回答。不要外推。"

**原理：** Claude 比 GPT 更诚实——但你需要主动告诉它"可以说不知道"。否则它会为了有用而"合理推测"。

---

### 💡 总结

和 Claude 交流就像和一流顾问合作：
- 交付质量 ≈ 输入的上下文质量
- 越具体的框架 → 越精准的输出
- 越明确的约束 → 越少的意外

**这 8 条不是理论，每条都是一个可以立刻复制的模板。** 挑 3 条最契合你场景的，明天就开始用。
