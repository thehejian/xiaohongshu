> 2026-06-02 · github.com/jackwener/opencli

每个网站一个 CLI 是什么体验 🤯

用过就回不去了 👇

**以前**你的一天是这样的：

- 查知乎热榜 → 打开网页 → 广告弹窗 → 关掉 → 找到热榜
- 想搜个 GitHub 项目 → 打开网页 → 搜关键词 → 看星数 → 翻 README → 半小时过去了
- 要发小红书 → 打开网页 → 登录 → 等加载 → 传图 → 写文案 → 选话题 → 发布 → 终于完了

每干一件事就要打开一个网页、等加载、找功能、点半天。

**现在呢？** 终端里敲一行：

```bash
opencli zhihu search AI Agent
opencli github search opencli
opencli bilibili search 飞书教程
opencli xiaohongshu publish '正文' --title '标题' --images 'xxx.png'
```

**148 个网站适配器，全部一行搞定。**

---

### 🎯 四大核心

**① 148+ 适配器 · 覆盖日常 90% 网站**

小红书、知乎、B站、微博、豆瓣、推特、ChatGPT、Claude、Gemini、GitHub、Docker、飞书CLI、企业微信、Obsidian、Notion、掘金…… 还在持续增加中。

**② Browser Bridge · 真浏览器自动化**

Chrome 驱动操作网页，登录态长期复用。不是截图模拟 —— 查 Feed、看笔记、搜内容、取评论、发帖子。所有你能想到的操作，全在终端完成。

**③ AI Agent 原生友好**

所有适配器支持 `--format yaml/json` 输出。Agent 拿到结构化数据，自动决策自动执行。本项目 20+ 个 lark skill 全是用它跑的。

**④ 13 个外部 CLI 一键聚合**

lark-cli · gh · vercel · wrangler · obsidian · tg · wx —— 一个 opencli 管所有命令行工具，浏览器和终端无缝衔接。
